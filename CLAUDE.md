# 制作方針

## 収集対象
- 速報・話題は、原則として「JSTにおいて前日に発生した・確認された出来事」を中心に収集する。
  - **配信日**（dateLong の日付）= 新聞を届ける日（JST当日）
  - **記事が扱う出来事の日**（ファイル名の YYYY-MM-DD）= 配信日の前日（JST昨日）
  - 配信日当日に起きることは「本日開幕予定」ではなく翌日の新聞で報じる
- 直近 7 日以内に同一内容を記録している場合は、内容が更新されない限り追加しない。
- 既に出ている話題を別の言い回しで繰り返すのではなく、実際に新しい事実がある場合だけ追記する。

## 日付・時制の厳格ルール（アーカイブ視点）

記事は「未来のある時点から振り返って読まれるアーカイブ」として書く。

### 禁止する相対表現

| 禁止表現 | 理由 | 代替表現 |
|----------|------|----------|
| 本日・今日 | 翌日以降に読むと意味がズレる | MM月DD日・YYYY-MM-DD |
| 明日・あした | 同上（未来を指すが即座に過去になる） | 具体的な日付 |
| 昨日・きのう | 同上 | 具体的な日付 |
| 前日（相対） | 「イベント前日」という意味では可、「昨日」の意味では禁止 | 具体日付 |
| 来週・来月 | 同上 | 具体的な週・月・日付 |

### badge フィールドの記法

- `"本日開始"` → `"MM/DD開始"` （例: `"6/5開始"`）
- `"前日"` → `"MM/DD開幕"` （例: `"6/6開幕"`）
- `"新発表"` / `"更新あり"` / `"開幕"` など時制に依存しない表現は使用可

### editions.yml の indexNote ルール

- `indexNote` は**前日（記事対象日）**に確認された出来事のみ列挙する
- 配信日（当日）のイベントを `indexNote` に書いてはならない
  - ❌ 「Hatsuboshi IDOL FESTIVAL**本日**開幕」（配信日=当日のイベント）
  - ✅ 「Hatsuboshi IDOL FESTIVAL **6/6・7**横浜アリーナ開幕（6/5確認）」

### 確認チェック

記事生成後、以下を必ず確認する：
1. `date -u -d '+9 hours' +%Y-%m-%d` でJST当日日付を取得する
2. **配信日** = JST当日、**記事対象日** = JST昨日 であることを確認する
3. ファイル名の日付が「JST昨日」であることを確認する
4. title・lead・本文に「本日/明日/昨日」等の相対表現がないことを確認する
5. badge に「本日〜」がないことを確認する
6. editions.yml の indexNote が配信日当日のイベントを参照していないことを確認する

## 記録ルール
- すでに掲載した内容の再録、要約の重複、同一ニュースの別ページ転載は避ける。
- 追記がある場合は「更新あり」と明示し、差分だけを記録する。

## 生成時の確認
- 記事の見出し・本文・メタ情報が、直近 1 週間の重複を含まないかを必ず確認する。
- もし同じ話題が複数回出ている場合は、最新の更新情報のみ残す。
- 出来る限り網羅的に、かつ重複のないように記録することを心がける。
- 1日の記事は最低10件以上を目指すが、内容の質を優先する。
- 書き出しは常に1記事ずつ行う。

## ファイル管理
- 記事ファイルは `docs/_posts/YYYY/MM/` のように年月サブディレクトリに格納する。
  例: `docs/_posts/2026/06/2026-06-05-cg-event.md`
- ファイル名の形式: `YYYY-MM-DD-BRAND-EVENT[-YYYYMMDD].md`
  例: `2026-06-05-cg-sousenkyo-20260605.md`
  複数日で継続する話題には日付サフィックスを付け、別記事として管理する。
- `edition` フィールドは廃止済み。frontmatter に書かない。

## editions.yml の更新ルール
- `dateLong` は必ずJST当日の日付を書く（`date -u -d '+9 hours' +%Y-%m-%d` で確認）。
- `pages` は「全 NN 面」（記事数に応じて調整）。
- `stamp` は「本日 06:00 締切」固定。
- 号数（issueNo）は `masthead.html` が `_config.yml` の `founding_date` から自動計算するため、editions.yml には書かない。
- `indexNote` には**前日（記事対象日）**の出来事のみ書く。配信日当日の予定イベントを書かない。

## 毎日の更新作業チェックリスト
1. `docs/_posts/YYYY/MM/` に記事ファイルを作成（10件以上目安）
2. `docs/_data/editions.yml` を更新（dateLong、pages、indexValue、indexNote）
3. `docs/_data/ranking.yml` を更新（8件のランキング）
4. `docs/_data/editorial.yml` を更新（社説タイトル・本文）
5. `python3 scripts/update_birthdays.py` を実行（birthdays.yml 自動更新）
6. コミット＆プッシュ

## 誕生日スクリプト
`scripts/update_birthdays.py` を実行すると `docs/_data/birthdays.yml` が自動更新される。
Claude Code が実行責務を持つ（GitHub Actions は未設定）。
